Understanding skins

The qnx.ui.skins package contains the basic implementations of the component skins. The skins classes provide a customizable, consistent, look and feel for the SDK component set.

The BlackBerry Tablet UI component set provides two distinct color themes: light and dark. The tablet uses the light themed component skins by default. The UI components vary in their complexity and in the way that the implement skinning. For example, a basic button component is simple to skin when compared to a Section Tile List, which is is comprised of many parts. The graphic assets that are used in skins take advantage of 9 slice scaling. This allows them to be resized with little or no distortion. Skins contain visual assets that are used to reflect state changes. These can be any type of Display Object. For example, a button may use multiple skin assets, where each asset reflects a button state (ie. down, up, disabled). These component states are defined in qnx.ui.skins.SkinStates

Skins derive from the qnx.ui.skins.UISkin class that registers skins for certain states and handles the state changes to show the appropriate skin associated with the state.

In order to create a custom skin, you must (at the very least) implement the qnx.ui.skins.ISkin interface. The quickest way to create a custom skin is to inherit from the qnx.ui.skins.UISkin class to take advantage of base functionality for handling state changes and registering skins. To create a custom skin, create a class that extends qnx.ui.skins.UISkin. To determine what assets are used for each state, they must be registered. To register assets, override the protected function initializeStates. You can then register your skin assets by associating an asset with an intended intended state.

In the following code listing, the initializeStates function registers existing assets to their states, with the assumption that these assets have been created/instantiated in the class prior to their registration: 

     
    override protected function initializeStates():void 
    {
	setSkinState(SkinStates.UP, upSkin );
	setSkinState( SkinStates.SELECTED, selectedSkin );
	setSkinState( SkinStates.DISABLED, disabledSkin );
	setSkinState( SkinStates.DOWN, downSkin );
	 showSkin( upSkin );
    }

In the example above, a reference to each of the skins ( upSkin, selectedSkin, disabledSkin, downSkin ) is being used to register against the state. Finally, showSkin(upSkin) is called to set the initial skin to show.

The following class illustrates how to make a custom skin for a basic button, with the assets being created and drawn using the drawing API: 

 
  package
  {
	import qnx.ui.skins.SkinAssets;
	import qnx.ui.skins.UISkin;
	import qnx.ui.skins.SkinStates;


	import flash.display.Sprite;

	
	public class CustomButtonSkin extends UISkin 
	{
		/**@private**/
		protected var upSkin:Sprite;
		/**@private**/
		protected var selectedSkin:Sprite;
		/**@private**/
		protected var disabledSkin:Sprite;
	    	 /**@private**/
		protected var downSkin:Sprite;


		/**
		 *Create a custom button skin
		 */
		public function CustomButtonSkin() 
		{
			super( );
		}

	  
		override protected function initializeStates():void 
		{

		upSkin = new Sprite();
		upSkin.graphics.beginFill(0xFF6600);
		upSkin.graphics.drawRect(0,0,200,200);
		upSkin.graphics.endFill();

		downSkin = new Sprite();
		downSkin.graphics.beginFill(0x333333);
		downSkin.graphics.drawRect(0,0,200,200);
		downSkin.graphics.endFill();

	    	disabledSkin = new Sprite();
		disabledSkin.graphics.beginFill(0xCC0000);
		disabledSkin.graphics.drawRect(0,0,200,200);
		disabledSkin.graphics.endFill();

	    	selectedSkin = new Sprite();
		selectedSkin.graphics.beginFill(0x000000);
		selectedSkin.graphics.drawRect(0,0,200,200);
		selectedSkin.graphics.endFill();


		setSkinState(SkinStates.UP, upSkin );
		setSkinState(SkinStates.SELECTED,selectedSkin );
		setSkinState( SkinStates.DISABLED, disabledSkin );
		setSkinState( SkinStates.DOWN, downSkin );
		showSkin( upSkin );

		}
	}
  }

Once a skin is created, the skin can be applied to the component using the methods exposed. For example, to apply a skin to a button, you can pass the skin class that you just created into the setSkin method: 

 

   		
myButton = new Button()
myButton.width = 100;
myButton.setSkin(CustomButtonSkin);
addChild(myButton);

For a background on the skin assets that are shipped with this SDK, see Skin assets.
