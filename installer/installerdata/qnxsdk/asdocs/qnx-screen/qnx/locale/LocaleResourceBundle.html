The LocaleResourceBundle class loads and parses localized String files.

The following parsing rules are supported:

  • Lines starting with "#" or "!" are not parsed and are considered comments.
  • Any whitespace on the left of the "=" symbol is stripped out and the resulting value is used as the key.
  • Any whitespace at the start of the value, to the right of the "=" symbol, is stripped out.
  • To include whitespace at the beginning of your value, add a "\" to the beginning of the String in order to escape it.
  • Any whitespace at the end of a value is preserved and is not stripped out.
  • You can combine multi-line values by adding a "\" at the end of the line.

Your String can also contain wildcard replacements, which are replaced when a String is returned. For example: wildcard=And a %1% and a %2% and a %3%

To replace the wildcards, add additional parameters to a getResource method call, such as LocaleManager.localeManager.getResource( "wildcard", 1,2,3 );

load()method 
public function load(path:String):void

Loads and parses a property file asynchronously at the specified path. An Event.COMPLETE event is fired after the file is loaded and parsed.

The following example shows a simple bundle loader routine that creates and sets up a resource bundle. 

        public function BundleLoader()
        {
        bundle = new LocaleResourceBundle();
        bundle.addEventListener(Event.COMPLETE, handleLoadComplete);
        }
        public function load():void
        {
        bundle.load("locale/en_US/resource.properties");
        }

Parameters

path:String — The path to the property file to be loaded.
